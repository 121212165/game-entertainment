## 1. 创建Supabase项目

### 1.1 注册并创建项目
1. 访问 [https://supabase.com](https://supabase.com)
2. 点击 "Start your project"
3. 使用GitHub账号登录（或注册新账号）
4. 点击 "New Project"
5. 填写项目信息：
   - **Name**: `si-le-ma-app`
   - **Database Password**: 设置一个强密码并保存
   - **Region**: 选择 `Northeast Asia (Seoul)` 或 `Southeast Asia (Singapore)`
   - **Pricing Plan**: 选择 Free tier
6. 等待项目创建完成（约2分钟）

### 1.2 获取API密钥
1. 进入项目Dashboard
2. 点击左侧菜单的 **Settings** → **API**
3. 复制以下信息并保存：
   - **Project URL**: 类似 `https://xxxxxxxxxxxxx.supabase.co`
   - **anon/public key**: 以 `eyJ...` 开头的长字符串

2. 应该能看到以下表：

## 3. 部署Edge Functions

### 3.1 安装Supabase CLI
在Windows上安装Supabase CLI：

```powershell
# 使用npm安装
npm install -g supabase

# 验证安装
supabase --version
```

### 3.2 配置环境变量
对于每个Edge Function，需要配置以下环境变量：

**邮件服务 (Resend)**:
1. 访问 [https://resend.com](https://resend.com) 并注册
2. 获取API Key
3. 在Supabase Dashboard中：
   - 点击 **Edge Functions**
   - 点击函数旁边的设置图标
   - 添加环境变量：
     - `RESEND_API_KEY`: `re_xxxxxx`
     - `SUPABASE_URL`: 你的项目URL
     - `SUPABASE_SERVICE_ROLE_KEY`: 在Settings → API中找到

### 3.3 部署Edge Functions

**方法1: 使用Supabase CLI**
```powershell
# 登录
supabase login

# 链接项目
cd C:\Users\lenovo\Desktop\silema\si-le-ma-app
supabase link --project-ref YOUR_PROJECT_REF

# 部署检查函数
supabase functions deploy check-missed-check-ins

# 部署邮件函数
supabase functions deploy send-notification-email
```

**方法2: 使用Supabase Dashboard**
1. 点击左侧菜单的 **Edge Functions**
2. 点击 "New Function"
3. 函数名: `check-missed-check-ins`
4. 复制 `edge-functions/check-missed-check-ins/index.ts` 的内容
5. 点击 "Save" 然后 "Deploy"
6. 重复步骤创建 `send-notification-email` 函数

## 5. 配置Cron Job

### 5.1 创建Cron Job
1. 在Supabase Dashboard中，点击 **Edge Functions**
2. 点击 "Cron Jobs" 标签
3. 点击 "New Cron Job"
4. 填写配置：
   - **Cron Expression**: `0 1 * * *`
   - **Function**: `check-missed-check-ins`
   - **HTTP Method**: POST
5. 点击 "Create"

**Cron表达式说明**:
- `0 1 * * *` 表示每天UTC时间凌晨1点执行
- 北京时间是UTC+8，所以是北京时间早上9点

如果想要其他时间：
- `0 2 * * *` = UTC 2:00 = 北京时间 10:00
- `0 0 * * *` = UTC 0:00 = 北京时间 8:00
- `0 23 * * *` = UTC 23:00 = 北京时间 7:00 (次日)

### 5.2 测试Cron Job
1. 在Cron Jobs列表中找到你的任务
2. 点击 "..." 菜单
3. 选择 "Invoke manually" 来测试触发

## ❓ 常见问题

### Q: 应用提示 "登录失败"
A: 检查 `local.properties` 中的 `supabase.url` 和 `supabase.anon.key` 是否正确

### Q: Edge Function执行失败
A:
1. 检查环境变量是否配置正确
2. 查看Function的Logs标签获取详细错误信息
3. 确保Service Role Key正确（在Dashboard的Settings → API中）

### Q: 没有收到邮件
A:
1. 确认Resend API Key有效
2. 检查紧急联系人邮箱是否正确
3. 在Resend Dashboard中查看发送记录
4. 注意：免费域名可能被邮件服务商标记为垃圾邮件

### Q: Cron Job没有执行
A:
1. 检查Cron表达式是否正确
2. 确认Edge Function已成功部署
3. 尝试手动触发测试
