### 第4分钟：快速开始（2分钟）

#### Step 1：创建Supabase项目（1分钟）

```
1. 访问 https://supabase.com
2. 点击 "Start your project"
3. 创建新项目：
   - Name: si-le-ma-app
   - Region: Northeast Asia (Seoul)
   - 等待2分钟项目创建完成
4. 获取API密钥：
   Settings → API → 复制 URL 和 anon key
```

#### Step 2：配置数据库（30秒）

```
1. 在Supabase Dashboard打开 SQL Editor
2. 创建新查询
3. 复制 database/schema.sql 的内容
4. 点击 "Run" 执行
5. 重复步骤执行 database/rls_policies.sql
✅ 数据库配置完成！
```

#### Step 3：运行Android应用（30秒）

```
1. 打开 Android Studio
2. 导入 android 目录
3. 编辑 local.properties：
   supabase.url=你的URL
   supabase.anon.key=你的anon key
4. 点击运行按钮 ▶️
✅ 应用启动成功！
```

## 💡 核心概念速查

### MVVM架构

```
Model（数据层）：
  - User.kt：用户数据模型
  - CheckIn.kt：签到数据模型
  → 只负责数据结构，不包含业务逻辑

View（视图层）：
  - SetupScreen.kt：设置界面
  - CheckInScreen.kt：签到界面
  → 只负责UI显示，不包含业务逻辑

ViewModel（视图模型）：
  - SetupViewModel.kt：设置业务逻辑
  - CheckInViewModel.kt：签到业务逻辑
  → 连接View和Model，处理业务逻辑

好处：
- 职责清晰，易于维护
- 便于测试
- AI更容易理解和生成
```

### Repository Pattern

```
问题：直接在ViewModel中访问Supabase
❌ 代码重复
❌ 难以测试
❌ 耦合度高

解决：使用Repository隔离数据访问
✅ 统一数据访问接口
✅ 便于单元测试
✅ 降低耦合度

示例：
UserRepository.kt：
  - createUser()
  - getCurrentUser()
  - checkIn()
  → 所有数据操作都在这里
```

### RLS（行级安全）

```
问题：如何确保用户只能看自己的数据？

传统方案：
在代码中检查权限
if (currentUser.id == data.userId) {
    return data
}
→ 容易遗漏，不安全

Supabase方案：
在数据库层面设置策略
CREATE POLICY "Users can view own data"
ON users FOR SELECT
USING (auth_id = auth.uid());
→ 数据库自动过滤，100%安全
```

### Anonymous Auth

```
问题：用户不想注册怎么办？

传统方案：
强制用户注册账号
→ 用户体验差，流失率高

Supabase方案：
匿名认证
1. 自动生成唯一ID
2. 用户无感知
3. 数据安全隔离
→ 用户体验好，留存率高
```

## ❓ 常见问题速查

### Q1: 编译错误怎么办？

```
Step 1：查看错误信息
红色文字显示具体错误

Step 2：同步Gradle
Android Studio → File → Sync Project with Gradle Files

Step 3：清理缓存
Android Studio → Build → Clean Project
                  → Rebuild Project

Step 4：检查配置
local.properties 中的URL和Key是否正确
```

### Q2: 无法连接Supabase？

```
检查清单：
✅ 网络连接正常
✅ URL格式正确（https://开头）
✅ API Key正确（anon key，不是service_role key）
✅ Android权限已开启（INTERNET）
✅ Supabase项目已启动（不是暂停状态）
```

### Q3: Edge Function执行失败？

```
Step 1：查看日志
Supabase Dashboard → Edge Functions → 选择函数 → Logs

Step 2：检查环境变量
RESEND_API_KEY 是否配置

Step 3：手动测试
在Dashboard中点击 "Invoke" 测试函数

Step 4：查看代码
index.ts 中的逻辑是否有语法错误
```
